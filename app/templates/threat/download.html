Linux — systemd Service

Requires Python 3.7+ and systemd. Run as root for full network visibility.

Option A — Auto Install (Recommended)
# Download and run the installer as root
curl -O {{ server_url }}/threat/download/script/linux
chmod +x install_linux.sh
sudo ./install_linux.sh
Download install_linux.sh
Option B — Manual
  1. Download the agent: pwnbroker_agent.py
  2. Install deps: pip3 install requests psutil
  3. Register: python3 pwnbroker_agent.py --register --no-verify-ssl
  4. Run: python3 pwnbroker_agent.py --no-verify-ssl
  5. For persistence, create a systemd service pointing to the script.
pwnbroker_agent.py

Windows — Task Scheduler

Requires Python 3.7+ from python.org. Run PowerShell as Administrator.

Option A — Auto Install (Recommended)
# In PowerShell (as Administrator)
Invoke-WebRequest "{{ server_url }}/threat/download/script/windows" -OutFile install_windows.ps1
.\install_windows.ps1
Download install_windows.ps1
Option B — Manual
  1. Download: pwnbroker_agent.py
  2. Open Command Prompt: pip install requests psutil
  3. Register: python pwnbroker_agent.py --register --no-verify-ssl
  4. Run: python pwnbroker_agent.py --no-verify-ssl
  5. Create a Task Scheduler entry to run at startup.
The agent uses --no-verify-ssl by default when using a self-signed certificate. Remove this flag after uploading a CA-signed cert in Settings → TLS/HTTPS.
pwnbroker_agent.py
Agent Capabilities
Sends every 60 seconds:
  • Hostname & IP address
  • CPU & memory utilisation
  • Running process list (names)
  • Active network connections (remote IPs)
Server checks:
  • New remote IPs vs AbuseIPDB
  • Results cached 24 hours
  • Alerts created for suspicious IPs
  • Alerts returned to agent on next heartbeat
Python requirements:
  • Python 3.7+
  • requests
  • psutil
  • No other dependencies
