Scan Remote Dependencies via SSH

PwnBroker will SSH into the target, locate all dependency lockfiles under the specified path, and check them against the OSV vulnerability database.

This target has no SSH credentials. Configure SSH on the target first.
Absolute path on the remote machine to search for lockfiles. Subdirectories are scanned recursively (node_modules, .git, venv excluded).
After scanning, PwnBroker will open a GitHub pull request for each vulnerable dependency that has a known fixed version, bumping it directly in the manifest. Requires a GitHub token with repo scope in Settings → Threat Intel APIs.
Supported Lockfiles
requirements.txt → PyPI
Pipfile.lock → PyPI
poetry.lock → PyPI
package.json → npm
go.mod → Go
Cargo.lock → crates.io
Gemfile.lock → RubyGems
composer.lock → Packagist/PHP
composer.json → Packagist/PHP (fallback)
GitHub Advisory Database — automatically enriches results with GHSA advisories (including PHP/Composer packages) when a GitHub token is configured in Settings → Threat Intel APIs.
